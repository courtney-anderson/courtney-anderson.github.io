Inline vs Inline Blocks

What does it mean to display inline vs inline blocks?

When a webpage is created in code, a coding language called HTML is used to lay out the content and structure of the page. Different elements that make up the page, such as the page header, title, article, and footer, are arranged in order in this HTML document. Elements are often nested within others, like a logo sitting inside a navigation bar.

An additional language, called CSS, can then be used to define the styling and behaviours of these elements. For example: their dimensions, colours, borders, padding, alignment, and position. In CSS, this is achieved by referencing an element, specifying a property for that element, and then defining that property, e.g. button {background-color: white}.

A key property in CSS is the display property, which determines the layout of an element and its interaction with surrounding elements. The default display behaviour of most elements is to act like a block, meaning each element starts on a new line and fills that line's width. This behaviour can be changed in CSS by defining the display property as display: inline, resulting in the element only occupying the space needed for its content, without pushing subsequent content to a new line.

However, if this inline element needs its dimensions further defined, this is when display: inline-block can be used. This enables a different width and height to be specified, rather than the element merely wrapping its contents.
